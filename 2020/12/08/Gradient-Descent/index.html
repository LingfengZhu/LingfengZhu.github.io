梯度下降算法以及自适应学习率的一些个人总结。

普通梯度下降

  • $\Theta$：参数集合，包括所有权重向量 $w$ 和偏置项 $b$；
  • $\alpha$：学习率；
  • $x^{(i)}$：样本 $i$ 的特征值；
  • $y^{(i)}$：样本 $i$ 的类别标签；
  • $J(\Theta | x, y)$：损失函数/目标函数；
  • $\nabla_{\Theta} J(\Theta)$：损失函数 $J$ 对 参数 $\Theta$ 求导；

Batch Gradient Descent

在每一轮 Eppch 的训练过程中，只进行一次处理，同时处理整个训练集的所有样本（一次性存入内存）计算一次损失函数的梯度，并用该梯度对模型参数进行一次更新，一轮 Epoch 中进行了一次更新：

优点

  • 若目标函数为凸函数，能够保证收敛到全局最优值；若目标函数为非凸函数，能够收敛到局部最优值；
  • 将大量数据视为矩阵直接进行向量化计算，向量化计算的效率高于使用循环计算；

缺点

  • 由于每轮迭代都需要在整个数据集上计算一次，所以批量梯度下降可能非常慢；
  • 训练数较多时，需要较大内存；
  • 每个 Epoch 会计算很多相似样本的梯度，这部分计算是冗余的（这些相似样本对模型效果的影响是一相似甚至一致的，可能只需计算其中一个就足够了）
  • 批量梯度下降不允许在线更新模型（例如新增一个样本，则需要全部重新训练）；

Stochastic Gradient Descent

在每一轮 Eppch 的训练过程中，进行 $sampleSize$ 次处理，每次处理一个样本，根据该样本计算损失函数的梯度，并用该梯度对模型参数进行更新。处理完所有样本后视为完成一轮 Epoch，一轮 Epoch 中进行了 $sampleSize$ 次更新：

优点

  • 算法收敛速度比 Batch Gradient Descent 快（省去了冗余计算）；
  • 可以在线更新模型（例如新增一个样本，则只需基于之前的模型针对该样本继续训练即可）；
  • 有几率跳出一个比较差的局部最优解而收敛到一个更好的局部最优解甚至是全局最优解；

缺点

  • 容易收敛到局部最优解，并且容易被困在鞍点；
  • 由于每次只处理一个样本，因此一个 Epoch 内需要进行很多次更新，可能会引入较多的 noise；

注意

还有一个更为极端的 Stochastic Gradient Descent 算法：在每一轮 Eppch 的训练过程中，只进行一次处理，随机抽取一个样本，根据该样本计算一次损失函数的梯度，并用该梯度对模型参数进行一次更新。处理完该样本后视为完成一轮 Epoch，一轮 Epoch 中进行了一次更新：

Mini-batch Gradient Descent

设定一个 $miniBatchSize$，将样本随机划分为多个 mini-batch。在每一轮 Eppch 的训练过程中，进行 $\frac{sampleSize}{miniBatchSize}$ 次处理，每次处理一个 mini-batch，根据该样本计算损失函数的梯度，并用该梯度对模型参数进行更新。处理完所有 mini-batch 后视为完成一轮 Epoch，一轮 Epoch 中进行了 $\frac{sampleSize}{miniBatchSize}$ 次更新：

其中 $m$ 表示 $miniBatchSize$。

优点

  1. VS SGD:
    • 更新次数比 SGD 少，因此引入的 noise 会比较少，收敛比较稳定；
    • 每次针对整个 mini-batch 计算梯度，利用到了向量化运算的优势；
  2. VS BGD:
    • 计算效率比 BGD 更高；
    • 内存占用比 BGD 更小；

是目前的主流方法。

普通梯度下降算法面临的挑战

  1. 每次的更新方向完全依赖于当前计算出的梯度，可能导致不稳定；
  2. 选择适当的学习率 $\alpha$ 较为困难：太小的学习率会导致收敛缓慢，而太大的学习率会造成较大的波动和震荡，妨碍收敛；
  3. 对于不同的参数应该考虑使用不同学习率：对于经常更新的参数（即梯度经常不为 0 的参数），我们已经积累了大量关于它的知识，不希望被单个样本影响太大，希望学习速率慢一些；对于偶尔更新的参数（即梯度经常等于 0 的参数），我们了解的信息太少，希望能从每个偶然出现的样本身上多学一些，即学习速率大一些；
  4. 鞍点：普通梯度下降容易陷入马鞍面，最后被困在鞍点；

Gradient Descent with Momentum

（将梯度下降找最优解比作小球向下滚到最低点）借用物理中的动量概念，模拟物体运动时的惯性：即更新的时候在一定程度上保留之前更新的方向（惯性），同时利用当前计算出来的梯度（加速度）微调最终的更新方向：使用指数加权平均（Exponentially Weighted Average）实现。这样一来，可以在一定程度上增加稳定性（防止因为当前梯度方向的改变较大而造成震荡），从而更快地学习，并且还有一定摆脱局部最优的能力：

其中 $\beta$ 为比例参数（替代了学习率），一般选择 $0.9 \sim 0.99$ 之间的某个数；$v_t$ 就是第 $t$ 步更新时的动量，而上一步时的动量 $v_{t-1}$ 对于这一步来说就是惯性；这一步计算出来的梯度 $\nabla_{\Theta} J(\Theta)$ 可以看做这一步施加的加速度。

算法会观察历史动量 $v_{t-1}$：历史动量方向与当前梯度 $\nabla_{\Theta} J(\Theta)$ 方向越是一致，速度就会越快；历史动量方向与当前梯度 $\nabla_{\Theta} J(\Theta)$ 方向越是不一致，速度就会越慢。

Nesterov Momentum

在上述算法的基础上考虑改进：在小球向下滚动的过程中，若小球能够提前知道在哪些地方坡面会上升，在遇到上升坡面之前，小球就开始减速，则就能更快地滚到最低点。类比到梯度下降中，就是在计算当前梯度时，不是计算 ”当前位置 $\Theta$“ 的梯度 $\nabla_{\Theta} J(\Theta)$，而是用 ”正常情况下，下一步应该会到达的位置“（用 $\Theta - \beta v_{t-1}$ 近似）的梯度 $\nabla_{\Theta}J\left(\Theta-\beta v_{t-1}\right)$ 替代：

核心思想：如果当前位置为 $\Theta$，则下一步到达的位置应当为 $\Theta-\beta v_t$，而由于此时 $v_t$ 还未被计算出来，因此用 $v_{t-1}$ 近似。这个近似的 ”前瞻/展望“ 位置 $\Theta-\beta v_{t-1}$ 就类似小球目前还未滚到的前方位置，因此在这个位置求梯度就能让小球提前知道前方的情况，从而加速算法的收敛。

自适应学习率优化算法

在迭代的过程中，根据梯度的计算情况，对学习率 $\alpha$ 自动进行一定的调整。下文中的所有 $\epsilon$ 都是作为平滑项，为了防止分母为 0 而存在，一般取值为 $10^{-8}$。

AdaGrad

对于经常更新的参数（即梯度经常不为 0 的参数），我们已经积累了大量关于它的知识，不希望被单个样本影响太大，希望学习速率慢一些；对于偶尔更新的参数（即梯度经常等于 0 的参数），我们了解的信息太少，希望能从每个偶然出现的样本身上多学一些，即学习速率大一些。因此，如果能够衡量 ”参数出现的频率“，我们就能相应地调整其对应的学习率。

假设向量 $\vec{g}_t$ 表示第 $t$ 轮时计算得到的梯度（即 $\vec g = \nabla_{\Theta} J(\Theta)$ ），它的分量 $g_{i,t}$ 表示第 $t$ 轮时第 $i$ 个参数 $\theta_i$ 的梯度；假设向量 $\vec{r}_t$ 表示第 $t$ 轮时参数的出现频率它的分量 $r_{i,t}$ 表示第 $t$ 轮时第 $i$ 个参数 $\theta_i$ 的出现频率。

RMSprop (Root Mean Square prop)

为了解决 AdaGrad 中 $\vec r$ 一直递增导致学习率递减趋向于 0 的问题，考虑使用指数加权平均（Exponentially Weighted Average）引入一个新的权重超参数 $\rho$ 来控制 $\vec r$ 的取值范围，用参数 $i$ ”到第 $t$ 轮为止梯度值的平方值的加权平均（而不是平方和）“ 来衡量该参数 $i$ ”到第 $t$ 轮为止的出现频率“：

注意

类似于之前的 Nesterov Momentum 算法，在 RMSprop 中也可以通过在“前瞻/展望位置”替换“当前位置”计算当前梯度，来实现带有 Nesterov Momentum 的 RMSprop 算法。即：用 $\vec g = \nabla_{\Theta}J\left(\Theta-\beta v_{t-1}\right)$ 替换原来的 $\vec g$ 即可。

Adam (Adaptive Moment Estimation)

Adam 算法可以看做结合了 momentum 算法（梯度一阶矩）和 RMSprop 算法（梯度二阶矩）的结合体。它利用梯度的一阶矩估计和二阶矩估计动态调整每个参数的学习率，同时对初期的矩估计进行了偏差修正。Adam的优点主要在于经过偏置校正后，每一次迭代学习率都有个确定范围，使得参数比较平稳。公式如下：

Algorithm problems that requiring subset / permutation / combination / path as output can be solved by using Backtracking method. I summarized a framework to solve all the backtracking problems in this post.

Framework

Main Factors

The process of solving a backtracking problem is actually a process of traversal a decision tree. One should focus on three main factors:

  1. Path: The decisions that you have already made.
  2. First: The index used to mark the current starting point.
  3. Choose List: A list of all the decisions that you can make in this time point.
  4. Termination Condition: The condition that there is no decision left in the choose list (i.e. reaches the leaf of the decision tree).

Main Framework

The main framework of backtracking algorithm can be seen as follows:

The main point is the recursive process within the for loop of chooses: make the choose before the recursion, undo the choose after the recursion.

Examples

LeetCode 77. Combination

Problem

Given two integers n and k, return all possible combinations of k numbers out of 1 … n.

Example

Solution

Using the backtracking framework, every time choose one integer from 1…n and add it to the combination, then recursively choose other integers.

We define a backtrack function named backtrack(k, first, curr) which takes the index of first element to add and a current combination as arguments.

  • If the current combination is done, we add the combination to the final output.

  • Otherwise, we iterate over the indexes i from first to the length of the entire sequence n.

    • Make the choice: Add integer i+1 into the current combination curr.
    • Recursive step: Proceed to add more integers into the combination : backtrack(k, i + 1, curr). Here we use i+1 instead of first+1 since integers between i+1 and first are used (added) in previous loops, for combination/subset we do not need to count them again.
    • Undo the choice: Backtrack by removing i+1 from curr.

LeetCode 78. Subsets

Problem

Given a set of distinct integers, nums, return all possible subsets (the power set).

Example

Solution

The method is very similar to the last example. Just notice that in this case, we have to iterate k from 1 to len(num), and for each k, we do the exact same thing as the last example.

For each k:

We define a backtrack function named backtrack(k, first, curr) which takes the index of first element to add and a current combination as arguments.

  • If the current combination is done, we add the combination to the final output.

  • Otherwise, we iterate over the indexes i from first to the length of the entire sequence n.

    • Make the choice: Add integer nums[i] into the current combination curr.
    • Recursive step: Proceed to add more integers into the combination : backtrack(k, i + 1, curr). Here we use i+1 instead of first+1 since integers between nums[i] and nums[first] are used (added) in previous loops, for combination/subset problem we do not need to count them again.
    • Undo the choice: Backtrack by removing nums[i] from curr.

LeetCode 46. Permutations

Problem

Given a collection of distinct integers, return all possible permutations.

Example

Solution

Notice that we can get every permutations by:

  • Swap positions of the first integer and every other integers once at a time;
  • For every swapping, fix the first position, for the other integers, do the same swapping recursively;
  • Stop when all the positions are fixed, nums at this time point is a permutation, add a copy of nums to the final output.

We define a backtrack function named backtrack(nums, first) which takes nums and the index of first element to swap as arguments.

  • If the current permutation is done, we add the permutation (a copy of current nums) to the final output.

  • Otherwise, we iterate over the indexes i from first to the length of the entire sequence n.

    • Make the choice: swap nums[first] and nums[i].
    • Recursive step: Proceed to perform more swapping : backtrack(nums, first+1). Here we use first+1 since this is a permutation problem, though integers between nums[i] and nums[first] are swapped in previous loops, we can still NOT ignore them in future swapping steps.
    • Undo the choice: Backtrack by putting nums[first] and nums[i] in their original position.

LeetCode 51. N Queens

problem

The n-queens puzzle is the problem of placing n queens on an n times n chessboard such that no two queens attack each other. (Queens in same row/column/diagonal can attack each other.)

Given an integer n, return all distinct solutions to the n-queens puzzle.

Each solution contains a distinct board configuration of the n-queens’ placement, where 'Q' and '.' both indicate a queen and an empty space respectively.

Example

Solution

Use the backtracking framework. Place queens in every possible position, recursively place the next queens until the end. Then backtrack.

  1. We process one row at a time: For each row, we will put one queen. This satisfy the row validity.

  2. Notice that if position [i,j] and position [a,b] are in the same “right-top to left-bottom” diagonal, then we must have: i + j == a + b. That is, positions in the same “right-top to left-bottom” diagonal must have the same “indexes sum”. Furthermore, it means one “indexes sum” uniquely corresponds to one “right-top to left-bottom” diagonal.

    Based on this conclusion, we can use a dictionary uphill_valid to check the “right-top to left-bottom” diagonal validity: uphill_valid[sum] == False means the “right-top to left-bottom” diagonal represented by sum is occupied. For example, if we put a queen in position [i,j], then we will set uphill_valid[i+j] == False, which means we can not put any queen in any position [a,b] that satisfies a+b == i+j.

  3. Similarly, if position [i,j] and position [a,b] are in the same “left-top to right-bottom” diagonal, then we must have: i - j == a - b. That is, positions in the same “left-top to right-bottom” diagonal must have the same “indexes difference”. Furthermore, it means one “indexes difference” uniquely corresponds to one “left-top to right-bottom” diagonal.

    Based on this conclusion, we can use a dictionary downhill_valid to check the “left-top to right-bottom” diagonal validity: downhill_valid[diff] == False means the “left-top to right-bottom” diagonal represented by diff is occupied. For example, if we put a queen in position [i,j], then we will set downhill_valid[i-j] == False, which means we can not put any queen in any position [a,b] that satisfies a-b == i-j.

  4. Finally, use a dictionary col_valid to check the column validity: If we put a queen in the ith column, we will set col_valid[i] == False, which means we can not put any queen in any position of the ith column.

We define a backtrack function named backtrack(first, path) which takes first, the index of current processing row and path, a list representing a solution (path[i] == j means in this solution position [i,j] (ith row, jth column) has a queen.) as arguments.

  • If the index of current processing row is larger than that of the final row, we add the path to the final output (Re-organize the path into the required output format).

  • Otherwise, we iterate over the column indexes i from 0 to n.

    • If all the three validities are satisfied:
      • Make the choice: Put one queen in the ith column of the processing row (i.e. the position [first,i]): Two steps, 1) adding the index to path and 2) update the three validities.
      • Recursive step: Proceed to next row : backtrack(first+1).
      • Undo the choice: Backtrack by taking the queen in position [first,i] away: Two steps, 1) removing the index from path and 2) update the three validities.
